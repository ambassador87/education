В то время, как git reset может откатить несколько коммитов подряд, с помощью следующей команды, можно отменить изменения отдельно взятого коммита:
```
git revert abc123 def456 ghj789
git revert HEAD~1
```
Но в этом случае, не происходит отката коммита, а даже напротив происходит добавление нового коммита для каждого отмененного, но в нем будут выполнены действия противоположные указанному коммиту, чтобы отменить его изменения. Внешний вид истории коммитов немного страдает из-за этого, так как ошибочный коммит сохраняется в истории плюс появляется исправляющий его, но если ветка общая, то это best practice.

На самом деле `revert` может отменять несколько подряд идущих коммитов:
```
git revert abc123..ghj789 --no-commit
```
Чтобы Git не плодил коммиты для каждого отмененного нужно использовать флаг `--no-commit` в этом случае изменения будут только добавлены в индекс, но не закоммичены. Таким образом все изменения окажутся в одном коммите.

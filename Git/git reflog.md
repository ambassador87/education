Показывает историю только по локальному репозиторию. С помощью этой команды можно увидеть такую информацию:
- перемещение HEAD (например, после reset --hard)
- коммиты
- слияния, удаления веток
- изменения тегов

Эта команда может быть очень полезна в следующем случае. Представим, что вы удалили последний коммит через `git reset --hard HEAD^`. В этом случае последний коммит исчезнет из истории коммитов и `git log` просто не отобразит отмененный коммит, чтобы к нему откатиться. Да, конечно, можно восстановить последний коммит с помощью 
`git reset ORIG_HEAD`, но возможно мы сделали несколько удалений подряд, тогда этот способ не поможет. 
Стоит понимать, что `git reset --hard` не удаляет коммит физически, а только отсоединяет коммит от дерева и он хранится в базе git какое-то время пока gc его не удалит. 
И git reflog отобразит хеш всех коммитов, которые не забрал garbage collector.

Но у reflog есть доступ только к записям, которые еще сохранились в истории (по умолчанию 90 дней). И эту историю можно почистить как самим reflog так и gc.
```
# Сценарий использования: Удаление старых записей
git reflog
git reflog expire --expire=7.days --all --verbose
git reflog

git reflog expire --expire=1.minutes --all –-verbose
git reflog

# запуск сборщика мусора Git
git gc
```

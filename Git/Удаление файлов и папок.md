
#### git rm
| **ВАЖНО**: git rm работает только с отслеживаемыми файлами.
Чтобы удалить **отслеживаемый** файл из репозитория, нужно удалить его и добавить изменения в индекс - это два действия. Но в Git есть одна команда для этого
```
# Удаляет несклько файлов
git rm file.txt static/css/style.js

# Удаление по маске
git rm *.log

# Удалит все файлы .tmp во всех папках рабочего каталога
git rm **/*.tmp

# Удалит папку со всем что в ней есть
git rm -r my_folder/
```

Если нам нужно удалить файл, который мы изменили и добавили в индекс, Git не позволит это сделать. Поэтому можно использовать флаг `-f`, чтобы удалить принудительно:
```
git rm -f my_file.txt
```



#### git clean
Эта команда удаляет **неотслеживаемые** файлы и не указанные в .gitignore. Это массовая операция, она не работает с конкретными именами файлов.
Поскольку удаление таких файлов невозможно отменить Git очень боится, что ты удалишь что-то лишнее, поэтому `git clean` никогда не работает без флагов. Она просто выдаст предупреждение.

```bash
# 1. Сначала ВСЕГДА проверяем, что именно будет удалено.
# Флаг -n (dry-run) - холостой прогон, покажет какие файлы будут удалены, без удаления.
git clean -n

# 2. Удаляем только неотслеживаемые ФАЙЛЫ (без папок).
# Без флага -f (force) гит откажется удалять что-либо из соображений безопасности.
git clean -f

# 3. Удаляем неотслеживаемые ФАЙЛЫ и ПАПКИ.
# Флаг -d (directories) позволяет "венику" заходить в подпапки и вычищать их целиком.
git clean -fd

# 4. Тотальная зачистка (Hardcore mode).
# Флаг -x заставляет Git игнорировать правила из .gitignore. 
# Это удалит скомпилированные файлы, библиотеки (node_modules, venv) и логи.
git clean -fx

# 5. Интерактивная чистка.
# Гит будет спрашивать про каждый файл: "Удалить этот? А этот?". 
# Полезно, если в куче мусора нужно оставить один важный лог-файл.
git clean -i
```

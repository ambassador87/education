Такой тип аутентификации предполагает наличие приватного ключа.
```bash
# Проверить наличие private key
ssh -T git@github.com
# ssh -- утилита для удаленного доступа к серверам
# -T -- ключ проверки подключения
# git@github.com -- тот самый сервер, куда мы стучимся для проверки

# Сгенерирует пару: приватный и публичный ключи
ssh-keygen -t ed25519 -C "my_email@mail.ru"
# -t ed25519 -- указывает тип алгоритма шифрования
# -C "my_email" -- комментарий
```

После создания приватного и публичного ключей они будут находиться в каталоге: `User/user/.ssh/`, если не был указан иной. 

- Открываем содержимое файла с суффиксом `.pub` и копируем
- Добавляем на GitHub (Settings->SSH and GPG keys) в New SSH Key

После добавления публичного ключа, можно клонировать репозитории, используя ссылку на репозиторий `ssh` вместо `https`. Далее для аутентификации не придется вводить логин, пароль при git pull, git push, т.к. аутентификация будет происходить автоматически.

Может случиться так, что в процессе работы, нужно будет срочно выполнить задачи, несвязанные 
с текущей работой:
- Переключиться на другую ветку для выполнения новой задачи
- Нам сказали, что другой программист запушил на сервер важные изменения, которые нужно стянуть  
а у нас незавершенная работа.

Так как наша работа еще не завершена, а проделанную работу мы не хотим терять, перед нами  
появляется выбор, как же поступить?
1. Оставить все как есть и просто переключиться на другую ветку. Или другими словами, оставить  
изменения в рабочем каталоге. Но если в другой ветке есть изменения в тех же строках, что и у меня,   
Git не позволит переключить ветку, чтобы не потерять изменения.
2. Добавить изменения в индекс и переключиться на другую ветку. Но индекс не привязан к ветке.  
И после переключения на другую ветку изменения в индексе переедут следом за мной и ошибочно могут  
быть закоммичены в другую ветку.
3. Выполнить коммит. Но выполнять коммит незавершенного кода это сама по себе плохая практика.

Проще говоря, все эти варианты нам не подходят. Есть вариант убрать эти изменения в тайник (stash)
```shell
git stash
git stash push
```
В стеш, по умолчанию, убираются только отслеживаемые файлы, если нужно добавить неотслеживаемые:
```shell
# Добавляет неотслеживаемые или вновь созданные файлы
git stash -u
```
Когда мы извлечем эти изменения из стеша, файл по прежнему будет untracked.

После этого Git позволит нам переключиться на другую ветку. Мы выполним другие задачи, снова  
вернемся в свою ветку и достанем наши незавершенные изменения из стеша:
```shell
git stash pop
```
После того, как достали из стеша изменения `git status` сможет их отобразить.

Есть возможность добавлять изменения в несколько стешей. Например, я внес изменения в файлы,   
но появилась другая задача в этом проекте в другом файле и не связанная с моей текущей работой.  
Я сохраняю текущие изменения в стеш и приступаю к выполнению другой работы. Затем у меня появляется  
новая срочная задача, несвязанная с двумя предыдущими. И снова изменения по текущей задаче я убираю  
в стеш и приступаю к новой работе. Затем мне нужно переключиться а первую задачу. Я в очередной раз  
убираю текущую (третью) задачу в стеш.

Чтобы посмотреть список сохраненных стешей:
```shell
git stash list
```
Вытащить из стеша можно двумя способами:
```shell
# Достает изменения из стеша и удаляет из списка
# первый из списка
git stash pop

# по индексу из списка
git stash pop stash@{1}

# Достает изменения из стеша, не удаляя из списка
# первый из списка
git stash apply

# по индексу из списка
git stash apply stash@{1}
```

Стешей может быть несколько и запомнить, в каком из них какие изменения сложно. Поэтому можно  
посмотреть какие изменения в каком из стешей хранятся:
```shell
# в самом первом стеше
git stash show -p

# в указанном стеше
git shash show -p stash@{2}
```
Но лучшей практикой будет добавлять комментарий к стешу и :
```shell
git stash -m "Незавершенная задача"
```
Чтобы удалить стеш:
```shell
# Удаляет первый из списка
git stash drop

# Удаляет указанный из списка
git stash drop stash@{2}

# Удалить все стеши из списка
git stash clear
```

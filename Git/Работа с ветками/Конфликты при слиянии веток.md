В процессе слияния двух веток могут возникать конфликты, если ваши изменения  
затрагивают те же строки, что и чужие изменения в целевой ветке, с которой происходит слияние.

В случае конфликта при слиянии веток, Git в самом файле добавляет блоки с маркерами  
в конфликтных строках, показывая изменения из вашей ветки и из целевой.
```
<<<<<<< HEAD
# Изменения в целевой ветке (куда вливают)
# т.к. мы переключаемся на нее перед мерджем
# Это может быть main/master
==============
# Изменения в исходной ветке, коммит которой вливают
# Это наша ветка, в которой мы работали.
>>>>>>>>> abc123 "Комментарий чужого коммита"
```

Если мы хотим принять изменения из вливаемой ветки, то используем флаг `--theirs`,  
т.к. мы находимся в целевой.
Если хотим оставить изменения целевой ветки, то используем флаг `--ours`:
```shell
git checkout --ours [<имя_файла>]
# имя файла не обязательный аргумент, 
# если его не указать будут приняты изменения во всех файлах с конфликтами

git checkout --theirs [<имя_файла>]
```
Далее нужно добавить изменения в индекс и сделать коммит
```shell
git add <имя_файла>
git commit
```

Если в процессе разрешения конфликта мы запутались, то можно сделать шаг назад и отменить слияние: 
```shell
git merge --abort
```

***СОВЕТ***  
Чтобы минимизировать конфликты, рекомендуется перед слиянием выполнять `git pull`

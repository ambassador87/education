**ВАЖНО!**  
Ветка это не последовательность коммитов, а указатель на коммит.   
В режиме Attached head HEAD - указатель на ветку: HEAD -> main/feature/bugFix -> коммит  
В режиме Detached head HEAD - указатель на коммит: HEAD -> коммит

### Виды веток
1. Main/Master — главная ветка, содержащая стабильную версию кода.
2. Feature-ветки — создаются для разработки новых функций.
3. Bugfix-ветки — для исправления ошибок.
4. Release-ветки — для подготовки релиза.
5. Hotfix-ветки — срочные исправления в продакшене.

```sh
# Создаёт новую ветку и переключается на неё
git checkout -b <имя_ветки> [<хеш_коммита>]
git switch -c <имя_ветки> [<хеш_коммита>] # -c -- create

# Создает новую ветку, но не переключается на неё
git branch <имя_ветки> <хеш_коммита>

# Переключиться на другую ветку, переключает HEAD на другую ветку
git checkout <имя_ветки>

# В режиме Detached Head переключает HEAD на целевой коммит.
# Указатель ветки остается на месте.
git checkout <хеш_коммита>/HEAD^^/HEAD~2/main^/bugFix~2

# Переключает на предыдущую ветку, что может быть удобным для переключения между 2 ветками
git checkout -
git switch -

# Новая команда, так же переключает между ветками
git switch feature/main
```

### git branch
```shell
# Отображает все локальные ветки
git branch

# Отображает все ветки, включая удаленные
git branch -a
# -a -- значит all, т.е. все ветки

# Отображает только удалённые ветки
git branch -r
# origin/master
# -r -- remotes (удалённые)

# Создает новую ветку, но не переключается на неё
git branch <имя_ветки>

# Создает новую ветку от конкретного коммита
git branch <имя_ветки> <хеш_коммита>
```

### Отправка веток на удаленный сервер
Чтобы запушить изменения в ветку используется команда `git push`:
```shell
# Пушит только текущую ветку
git push

# Пушит все локальные ветки в удаленный репозиторий
git push --all origin

# Пушит только указанные ветки в удалённый репозиторий
git push origin feature/log-handling feature/registration
```

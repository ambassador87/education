
Stage Area (индекс) - область, где находятся файлы, подготовленные к коммиту. Для добавления файлов в индекс есть команда:
```bash
git add <file_name_1> <file_name_2> ... 
git add /<dir_name>
```
Репозиторий это место куда попадают файлы в текущем состоянии после коммита. Для выполнения коммита:
```bash
git commit
git commit -m "Здесь указывается комментарий к комиту"
```
Можно сразу сделать комит всех измененных и отслеживаемых файлов одной командой
```bash
git commit -a # то же что и последовательное git add, git commit
git commit -am "Комментарий к комиту"
```
Если к последнему отправленному комиту нужно добавить файлы, которые не были добавлены, их можно дописать в последний комит командой
```bash
# Добавляет изменения в последний отправленный коммит
git commit --amend -m "Комментарий"

# Если у коммита уже есть комментарий, можно оставить его без изменений
git commit --amend --no-edit
```
Чтобы посмотреть информацию о комите есть команда:
```bash
# показать детальную информацию о последнем коммите 
git show 
# показать детальную информацию об указанном коммите 
git show <commit_id> 
# вывести только имена файлов, участвовавших в коммите 
git show --name-only <commit_id>
```
Помимо лаконичного комментария к комиту можно добавлять подробные пояснения. Для этого нужно соблюсти:
- Выполнить git commit без флагов и тогда для добавления комментария будет запущен текстовый редактор. По умолчанию это vim. 
- В первой строке можно указать лаконичный комментарий, который будет отображаться при выводе в git show
- Затем нужно отступить одну пустую строку и написать подробное пояснение к комиту

### Рекомендации по оформлению сообщений для комита
- Пустая строка между темой сообщения и его телом
- Тема сообщения должна быть короче 50 символов
- Тема сообщения должна быть написана в верхнем регистре (Caps Lock) - опционально
- Точка не ставится в конце темы
- Тело сообщения должно быть короче 80 символов
- Тело сообщения должно объяснять: что изменено, а не почему были сделаны изменения
- Тема сообщения должна отвечать на вопрос: "Что делает коммит?"

### Пример комита согласно конвенции оформления комитов

**УДАЛИТЬ УСТАРЕВШИЕ ФУНКЦИИ АВТОРИЗАЦИИ**

Стереть методы login_old() и validate_v1() из контроллера доступа.

---
### Почему это хороший пример:

- **Тема (38 символов):** Написана капсом, отвечает на вопрос «Что делает коммит?» (Удаляет функции).    
- **Пунктуация:** В конце заголовка нет лишних точек.    
- **Тело (64 символа):** Конкретно описывает, какие именно методы были удалены из проекта.   
- **Формат:** Между темой и телом соблюдена пустая строка.
